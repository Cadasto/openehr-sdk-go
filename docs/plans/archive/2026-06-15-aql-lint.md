## Goal

Ship a **building-block** parse + lint pipeline for hand-written, imported, or `NewQuery(literal)` AQL ‚Äî without replacing the typed builders (REQ-055) or the CDR as semantic authority at execute time (PROBE-021).

Consumers: CI validators in clinical-modelling repos, MCP tools linting stored queries before upload, pre-flight checks on `ExecuteString` payloads, and `validation.ValidateAQL(q, compiledOPT)` for template-aware archetype/path checks.

**Non-goals:** Accept every syntactically valid AQL 1.1.0 query the CDR would run; reject only what the lint contract defines. Syntax floor is the **SDK-maintained grammar profile** (foundation AQL 1.1.0 + documented corrections), not a live pull from specifications.openehr.org on every build.

1. **Drop corrected** `AqlLexer.g4` and `AqlParser.g4` into `resources/aql/grammar/active/` (or open a PR with them).
2. **Populate `baseline/`** once from [specifications-QUERY tag `Release-1.1.0`](https://github.com/openEHR/specifications-QUERY/tree/Release-1.1.0/docs/AQL/grammar) if not already present.
3. **Write `DIVERGENCES.md`** ‚Äî one `SDK-AQL-NNN` block per accepted delta (five entries; see ¬ß SDK grammar profile).
4. **Add one minimal `.aql` file per divergence** under `openehr/aql/parse/testdata/grammar/` proving the fix (parse succeeds or fails as intended).

Optional maintainer tooling (Phase 0): `scripts/aql-grammar-diff.sh` prints a unified diff `baseline/` ‚Üí `active/` for PR review ‚Äî not required for v1.

Mirror [ADR 0001](../../adr/0001-bmm-version-bump-runbook.md) spirit:

1. Import new upstream `.g4` into `baseline/` (new `PIN`).
2. Re-apply or drop each `SDK-AQL-NNN` fix: if upstream absorbed the fix, remove the divergence row and merge `active/` toward baseline.
3. Re-run `make aqlgen` + `make aqlgen-verify` + full `openehr/aql/parse/...` tests.
4. Note grammar profile bump in CHANGELOG (artefact class: ‚ÄúAQL grammar profile‚Äù).

## SDK grammar profile

**Status:** Locked for Phase 0  
**Baseline:** QUERY Release-1.1.0 `AqlLexer.g4` / `AqlParser.g4` (frozen under `resources/aql/grammar/baseline/`).  
**Active:** `baseline/` plus the five deltas below (`resources/aql/grammar/active/`). Fork review is complete; only accepted changes are listed here. Rejected fork patterns are not re-litigated ‚Äî optional one-liners may appear in `DIVERGENCES.md` ¬ß Considered and rejected.

### Design rationale ‚Äî VERSION containment

Version predicates (`latest_version()`, `all_versions()`) belong on a **`VERSION` containment operand**, not on clinical RM classes (`OBSERVATION o[all versions()]`). The data tree (`COMPOSITION` ‚Üí `OBSERVATION` ‚Ä¶) is separate from version machinery (`VERSIONED_OBJECT` ‚Üí `VERSION` ‚Üí snapshot). Projecting `v/commit_audit/‚Ä¶` or ranging history requires `CONTAINS VERSION v[‚Ä¶]` (or `FROM VERSION v[‚Ä¶]`). The SDK profile **keeps the foundation grammar** for this ‚Äî no `SDK-AQL-NNN` row.

**Lint / builder:** `OBSERVATION o[all versions()]` is a syntax error; REQ-055 emitters **SHOULD** add explicit `VERSION` + predicate when version metadata or history is needed (emit `[latest_version]` / `[all_versions]` explicitly until SPECPR-481 lands).

**Regression:** `version_on_class.reject` must not parse; `version_containment.aql` (`CONTAINS VERSION v[all_versions] CONTAINS COMPOSITION c`) must parse.

### Accepted deltas (`SDK-AQL-NNN`)

| ID | Area | Change from baseline | Regression fixture(s) |
|---|---|---|---|
| **SDK-AQL-001** | Lexer ‚Äî string fn vs `CONTAINS` operator | Remove `CONTAINS` from `STRING_FUNCTION_ID`; add `CONTAINS_STR: C O N T A I N S '_' S T R` (longest-match safe vs containment `CONTAINS`; fallback `STR_CONTAINS` if `aqlgen` fails). Spec spelling `CONTAINS(expr, sub)` **not** accepted ‚Äî use `CONTAINS_STR(expr, sub)`. | `contains_containment.aql`, `contains_str_fn.aql`, `contains_spec_fn.reject` |
| **SDK-AQL-002** | Parser ‚Äî `selectExpr` | Add `SYM_ASTERISK` (`SELECT *`). | `select_star.aql` |
| **SDK-AQL-003** | Parser ‚Äî `limitClause` | `limitOperand` = `INTEGER \| PARAMETER` for `LIMIT` / `OFFSET`; **no** reversed `OFFSET ‚Ä¶ LIMIT` order. | `limit_offset_param.aql`, `offset_before_limit.reject` |
| **SDK-AQL-004** | Lexer ‚Äî `ESCAPE_SEQ` | Extend escapes with `\\"` and `\\*`. | `escape_quote_star.aql` (or inline in string-literal syntax tests) |
| **SDK-AQL-005** | Lexer ‚Äî comments | `COMMENT ‚Üí skip` (not `COMMENT_CHANNEL`). | ‚Äî (parse-only; no behavioural test required) |

**SDK-AQL-002 rationale ‚Äî `SELECT *` is an intentional relaxation (settled).** Official openEHR AQL has no bare `SELECT *` (projection columns must be explicit). The SDK profile **deliberately admits it**: lint is a pre-flight aid, not a spec-conformance gate, and `SELECT *` is a common convenience some CDRs honour. Consequence: a query the SDK lints clean may still be rejected by a strict CDR ‚Äî consistent with the plan's "pass lint ‚âÝ execute success" stance, so this relaxation is safe. Classify SDK-AQL-002 in `DIVERGENCES.md` as a **relaxation** (admits *more* than official AQL) ‚Äî distinct from SDK-AQL-001/003/004 **corrections** (which fix foundation-grammar bugs); add an `Upstream status: intentional-relaxation` value for relaxation rows. REQ-109 prose must state this so consumers never read "lint-clean" as "spec-conformant". (`COUNT(*)` is separate and spec-valid regardless.)

**SDK-AQL-001 lexer sketch:**

```antlr
CONTAINS_STR: C O N T A I N S '_' S T R ;
STRING_FUNCTION_ID: LENGTH | CONTAINS_STR | POSITION | SUBSTRING | CONCAT_WS | CONCAT ;
CONTAINS : C O N T A I N S ;   // containment operator ‚Äî unchanged
```

**Phase 0:** Copy baseline ‚Üí `active/`, apply the five deltas, mirror each row in `DIVERGENCES.md`, add fixtures under `openehr/aql/parse/testdata/grammar/`, run `make aqlgen-verify`.

### Baseline fidelity (no `SDK-AQL` row ‚Äî foundation grammar unchanged)

- `VERSION` class operand + `versionPredicate`; full `pathPredicate` on `identifiedPath` / `pathPart`
- Standalone `[$param]` in predicates; `ORDER BY` long and short keywords; extension `IDENTIFIER(...)` functions
- Deprecated `TOP` in `selectClause` (foundation grammar retained; REQ-055 does not emit it)
- Compact and punctuated ISO8601 literals; `SYM_DOUBLE_DASH? EOF` on `selectQuery`; split `AqlLexer` / `AqlParser` for codegen

### Package boundaries

| Package | Responsibility | Imports |
|---|---|---|
| `openehr/aql/parse/` | `Parse(q string) (*Document, error)` ‚Äî syntax only; positions for diagnostics | Generated ANTLR listeners + thin hand-written AST |
| `openehr/aql/lint/` | `Lint(doc, *LintOptions) Result` ‚Äî semantic subset (paths, archetypes, parameters) | `parse/`, `openehr/template` (path grammar only), optionally `internal/templatecompile` via validation seam |
| `openehr/aql/` | Existing `Query`, builders, `NewQuery` ‚Äî unchanged wire contract | May call `parse` for optional `Query.Lint()` helper |
| `openehr/validation/` | `ValidateAQL(q aql.Query, c *templatecompile.Compiled) Result` ‚Äî bridges to shared `Issue` model (REQ-102) | `aql/lint/`, `internal/templatecompile` |

**REQ-013:** `openehr/aql/parse/` and `openehr/aql/lint/` **MUST NOT** import `transport/`, `auth/`, `openehr/client/*`, or `openehr/serialize/`. `lint` **MAY** import `openehr/template` for path validation (same as validation today).

**Codegen:** `internal/aqlgen/` + `cmd/aqlgen/` (or extend Makefile like `bmmgen`) ‚Äî **not** public API. Generated parsers live in `openehr/aql/parse/internal/antlr/` or `openehr/aql/parse/gen/` (pick one; mirror `bmmgen` ‚Äúgenerated subtree‚Äù pattern).

**Dependency:** add `github.com/antlr4-go/antlr/v4` (the current ANTLR Go *runtime* module is major v4 ‚Äî there is no `‚Ä¶/antlr/v2`; the plan's earlier `v2` was a typo) ‚Äî first non-OTel runtime dep, justified for spec fidelity; document in ADR 0007 and `docs/architecture.md` dependencies table.

**Codegen toolchain ‚Äî containerised generator, mirrors the existing `LINT_IMAGE` shim.** The ANTLR *runtime* (`antlr4-go/antlr/v4`) is pure Go; only the *generator* (`.g4` ‚Üí Go) is Java. Confine Java to a transient codegen container so neither the host nor the Go dev image needs a JRE ‚Äî the same move the repo already makes for golangci-lint ([`Makefile`](../../../Makefile) `GOLANGCI` shim + [`Dockerfile`](../../../Dockerfile) `golangci` stage). Add a pinned `antlr` stage (`FROM eclipse-temurin:<v>-jre` + checksum-pinned `antlr-<v>-complete.jar`) beside it, and run it exactly like the lint shim:

```make
ANTLR_IMAGE ?= openehr-sdk/antlr:4.13.2   # built from the Dockerfile antlr stage
aqlgen:        ## regenerate the AQL parser (maintainer-only; needs Docker, not a host JRE)
	docker run --rm --user $$(id -u):$$(id -g) -v "$(PWD)":/work -w /work $(ANTLR_IMAGE) \
	  -Dlanguage=Go -o openehr/aql/parse/gen -package gen \
	  resources/aql/grammar/active/AqlLexer.g4 resources/aql/grammar/active/AqlParser.g4
aqlgen-verify: ## fail if the committed parser drifts from active/ (CI needs only Docker)
	# regenerate into a tmp dir, diff against openehr/aql/parse/gen/, fail on drift
```

The generated Go is **committed**; `build`, `test`, and `make ci` never invoke the container ‚Äî they compile the committed parser against the pure-Go runtime. Java is touched only by `make aqlgen` (maintainer, on grammar change) and optionally `make aqlgen-verify` in CI (which needs **Docker**, already present ‚Äî not a host JRE).

**Version lockstep (the one real gotcha):** the ANTLR *tool* version and the `antlr4-go/antlr/v4` *runtime* version are released together; a mismatch generates Go that will not compile against the runtime. Bump `ANTLR_IMAGE` and the go.mod runtime in the same change; record both in `resources/aql/grammar/README.md` PIN and ADR 0007.

## Lint contract (normative target for REQ-109)

Lint **MUST** run collect-all (same spirit as REQ-102): return every issue, not fail-fast.

### Layer 1 ‚Äî Syntax (grammar-backed)

- Input **MUST** parse as `selectQuery` per **`resources/aql/grammar/active/`** (SDK grammar profile).
- On failure: issue code `aql_syntax`, sentinel `aql.ErrSyntax` (new), `Issue.Path` empty, `Detail` carries ANTLR line/column.
- Empty / whitespace-only: `aql_empty` before parse (today `Query.Validate` only).

### Layer 2 ‚Äî Shape (AST walk, no CDR)

| Check | Issue code | Rule |
|---|---|---|
| `SELECT` present with ‚â•1 projection | `aql_select` | |
| `FROM` present | `aql_from` | |
| `FROM`/`CONTAINS` tree has ‚â•1 `archetypePredicate` or `VERSION`/`EHR` entry with identifiable scope | `aql_from_archetype` | v1: at least one `openEHR-EHR-‚Ä¶` archetype HRID or `$param` in a predicate position |
| `$name` placeholders in `Q` **MUST** have matching keys in `Query.Parameters` (when linting `aql.Query`) | `aql_unbound_param` | Complements REQ-055 injection story |
| Parameters in map **MUST** appear in `Q` (warning or error ‚Äî pick error for v1) | `aql_unused_param` | Optional v1.1 |

### Layer 3 ‚Äî Path & template (only when `*templatecompile.Compiled` provided)

| Check | Issue code | Rule |
|---|---|---|
| Each `identifiedPath` in `SELECT` / `WHERE` (excluding alias roots) **MUST** use path segments parseable by the SDK‚Äôs **AQL path normaliser** (new: strip alias, map to `template.Path` or dedicated `aql.Path` type) | `aql_path_syntax` | Do **not** reuse OPT `NodeAt` directly ‚Äî AQL paths are alias-qualified |
| When `c != nil`: each archetype HRID in `FROM`/`CONTAINS` **MUST** appear in `c.AllByArchetypeID()` (or index TBD on `Compiled`) | `aql_archetype_not_in_template` | From archived validation plan |
| When `c != nil`: normalised paths **SHOULD** resolve to a node in `c` (exact `NodeAt` on compiled AQL paths) | `aql_path_not_in_template` | **Warning** in v1 ‚Äî CDR may still resolve paths the OPT compiler does not index (document false-positive policy) |

**Explicitly out of scope (v1):** terminology `TERMINOLOGY()`, function signatures, `MATCHES` value-set membership, version predicates beyond parse, `ORDER BY` type checking, execute-time `ErrPathResolution` (PROBE-021).

## Concrete interfaces (load-bearing types)

These are the public shapes Phases 1‚Äì3 must land. They are decided here ‚Äî not discovered mid-build ‚Äî because the AST, the lint surface, and AQL-path handling are the contracts every later task depends on. Field sets are a **starting sketch**; extend per the grammar labels actually needed, but keep the generated ANTLR types unexported behind these.

### Phase 1 ‚Äî `openehr/aql/parse` AST

```go
// Document is the decorated, generated-type-free AST root.
type Document struct {
    Select  SelectClause
    From    FromExpr
    Where   *Predicate   // nil when absent
    OrderBy []OrderTerm
    Limit   *int         // paging stays out of the lint contract; surfaced for completeness
    Offset  *int
    Pos     Position     // for diagnostics
}

type SelectClause struct {
    Distinct bool
    Fields   []SelectField // ‚â•1; path, alias, literal, or function call
}
type SelectField struct {
    Path *IdentifiedPath // nil for literal/function projections
    Func *FuncCall
    AS   string          // optional column alias
    Pos  Position
}
type FromExpr struct {
    Root     ClassExpr
    Contains *Containment // nil for a bare FROM
}
type Containment struct { // CONTAINS tree; siblings via Op, nesting via Children
    Operand  ClassExpr
    Op       string       // "", "AND", "OR"
    Children []Containment
    Negated  bool         // NOT CONTAINS
}
type ClassExpr struct {
    RMType    string
    Alias     string         // "" when anonymous
    Archetype string         // "" or openEHR-‚Ä¶; archetype predicate
    Predicate *PathPredicate // standing predicate, e.g. [ehr_id/value=$x] or version predicate
    Pos       Position
}
type IdentifiedPath struct {
    Alias    string         // root binding; MUST resolve to a FROM/CONTAINS alias
    Segments []PathSegment  // /data[at0001]/events[at0006]/‚Ä¶
    Pos      Position
}
type Predicate struct { /* WHERE tree: comparison | AND/OR/NOT | EXISTS | MATCHES | LIKE */ }

type Position struct{ Line, Col, Offset int }
```

`parse.Parse(q string) (*Document, error)` wraps `AqlParser.selectQuery`; on failure it returns a `*parse.SyntaxError` (carrying `Position`) that wraps `aql.ErrSyntax`.

### Phase 3 ‚Äî lint surface and the Issue-model decision

**Decision (was the Phase 3 "parallel shape or convert" either/or ‚Äî resolve it now):** `lint` returns its **own** `lint.Result` / `lint.Issue` and does **not** import `openehr/validation`, so the building block stays free of the validation package. `validation.ValidateAQL` converts `lint.Issue` ‚Üí the shared `validation.Issue` (REQ-102) at the bridge. Dependency arrow is `validation ‚Üí aql/lint`, never the reverse.

```go
// openehr/aql/lint
type Severity int
const ( Error Severity = iota; Warning )

type Issue struct {
    Code     string   // "aql_syntax", "aql_from_archetype", "aql_path_not_in_template", ‚Ä¶
    Path     string   // AQL path concerned; "" when not path-scoped
    Detail   string   // human-readable; ANTLR line/col for syntax
    Severity Severity // aql_path_not_in_template is Warning in v1
}
type Result struct{ Issues []Issue }
func (r Result) OK() bool // true when no Error-severity issue

type Options struct {
    Compiled *templatecompile.Compiled // nil ‚Üí Layers 1‚Äì2 only; non-nil ‚Üí Layer 3
    Query    *aql.Query                // optional ‚Üí enables aql_unbound_param / aql_unused_param
}
func Lint(doc *parse.Document, opts *Options) Result
```

**Sentinel reconciliation (the plan currently names two):** `aql.ErrSyntax` (new, `openehr/aql/errors.go`) is the **parse-layer** sentinel. `validation.ErrAQLSyntax` **already exists** ‚Äî reserved today in `openehr/validation/errors.go`. Keep both: `parse`/`lint` consumers match `aql.ErrSyntax`; the `validation` bridge maps a Layer-1 issue to `validation.ErrAQLSyntax` via `Issue.Err()`. Phase 1 task 4 / Phase 3 task 4 must state this so neither sentinel is duplicated or dropped.

### Phase 2 ‚Äî AQL path normalisation (highest-risk piece ‚Äî spike first)

AQL paths are **alias-qualified** (`o/data[at0001]/‚Ä¶`) and are **not** directly consumable by OPT `NodeAt`, which keys on compiled, alias-free AQL paths. The normaliser is the load-bearing unknown; build a throwaway spike against the PROBE-020 golden + `vital_signs.opt` before committing the API.

```go
// openehr/aql (small path helper) or openehr/aql/lint
type Path struct {
    Alias    string        // stripped root binding
    Segments []PathSegment // attribute + optional [predicate]
}
// Normalise strips the alias and yields the segment list to match a compiled template.
func Normalise(p parse.IdentifiedPath) (Path, error)
```

Layer-3 resolution: resolve the alias to its bound `ClassExpr` (archetype root), then exact-match the remaining segments against `Compiled.NodeAt` on the archetype-rooted AQL path ‚Äî `NodeAt` already exists and does exactly this. For `aql_archetype_not_in_template` (HRID membership), the data is already present per node via `CompiledNode.ArchetypeID()`; the lint layer either walks the tree once or you add a one-line `Compiled.AllByArchetypeID()` helper next to the existing `AllByRMType`/`AllByNodeID`. No deep template work is required. `aql_path_not_in_template` stays a **warning** in v1 (a CDR may resolve paths the OPT compiler does not index).

### Phase 1 ‚Äî Public parse API + syntax errors

**Outcome:** `openehr/aql/parse.Parse` returns a stable, hand-written AST (decorates ANTLR tree); positions for diagnostics.

**Tasks:**

1. **AST types** ‚Äî `Document`, `SelectClause`, `FromExpr`, `ContainsExpr`, `IdentifiedPath`, `Predicate`, ‚Ä¶ (subset of grammar labels needed for lint + future format).
2. **Visitor** ‚Äî ANTLR parse tree ‚Üí AST; hide generated types from public API.
3. **`parse.Parse(string) (*Document, error)`** ‚Äî wrap `AqlParser.selectQuery`; export `parse.SyntaxError` with offset/line/col.
4. **`aql.ErrSyntax`** in `openehr/aql/errors.go`; `errors.Is` contract.
5. **Tests** ‚Äî table-driven valid/invalid strings in `openehr/aql/parse/testdata/` only (grammar regressions + hand-crafted syntax cases). **No** import of openEHR-antlr4 fixtures.

**Definition of done:** `go test ./openehr/aql/parse/...` green; golden AQL from REQ-055 parses.

## Open decisions (resolve in Phase 0 ADR)

1. **Canonical spec home** ‚Äî `wire.md` ¬ß REQ-109 vs `clinical-modeling.md` (recommend **wire.md** next to REQ-055). Note: `clinical-modeling.md` already carries the `ValidateAQL` / `ErrAQLSyntax` placeholder (¬ß REQ-102 deferred list) ‚Äî wherever REQ-109 lands, that placeholder must be updated to point at it (Phase 3 task 5).
2. **Generated code path** ‚Äî `parse/gen/` vs `parse/internal/antlr/` (recommend **`parse/gen/`**).
3. **`aql_path_not_in_template` severity** ‚Äî error vs warning (recommend **warning** v1).
4. **PROBE ID** ‚Äî ‚úÖ **confirmed free**: `PROBE-028` (and `PROBE-029`) are unused in `conformance.md` (highest reserved 022‚Äì027, 030, then 040+). `REQ-109` is free in `REQ.md`/`traceability.yaml`; `ADR 0007` is the next free ADR slot.
5. **Grammar profile** ‚Äî locked in ¬ß SDK grammar profile; `DIVERGENCES.md` mirrors `SDK-AQL-001` ‚Ä¶ `005`.
6. **`SELECT *` (SDK-AQL-002)** ‚Äî ‚úÖ **resolved: keep as an intentional relaxation** (rationale in ¬ß SDK grammar profile; classified as a relaxation, not a correction, in `DIVERGENCES.md`).
7. **`Compiled` archetype-HRID collection (minor)** ‚Äî Layer-3 path resolution already has what it needs: `Compiled.NodeAt(path)` resolves a compiled AQL path, and each `CompiledNode.ArchetypeID()` carries its HRID. The only missing convenience is "list all archetype HRIDs in the template" for `aql_archetype_not_in_template` ‚Äî either walk the tree once in the lint layer or add a one-line `Compiled.AllByArchetypeID()` helper (mirrors the existing `AllByRMType` / `AllByNodeID`). Implementation-time choice, not a blocker.
