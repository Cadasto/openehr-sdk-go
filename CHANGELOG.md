# Changelog

All notable changes to `github.com/cadasto/openehr-sdk-go` are recorded here.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Entries under `## [Unreleased]` are **short and high-level**: one-line bullets naming the artefact class and scope. File-level detail belongs in commit messages and PR bodies — see `AGENTS.md` > Code style and conventions.

## [Unreleased]

### Added

- Repository scaffolding: module layout, build tooling (Makefile + Docker `dev` stage), AI-assistant documentation set (`AGENTS.md`, `.claude/CLAUDE.md`, `docs/architecture.md`, `docs/ai-workflow.md`), and CI workflow (`.github/workflows/ci.yml` + `docs/ci.md` + `make ci`).
- Normative specifications tree under `specs/`: REQ/PROBE/STRAND identifiers; module layout, idiom, RM modeling, BMM conformance (REQ-041..047), auth, wire format, service discovery, conformance probes, use cases, research strands; Cadasto-platform extensions (canonical XML REQ-056, stored AQL REQ-057, Datamap V2 REQ-058, tenant binding REQ-065, AI caller REQ-066, platform principal REQ-067, SMART REQ-068, TLS REQ-092, openEHR header family REQ-059, error envelope REQ-093, `Prefer` REQ-094, OpenAPI source REQ-095).
- Implementation plans under `docs/plans/`: BMM codegen, canonical JSON / XML serialization, REST API client.
- Pinned BMM schemas under `resources/bmm/` (base 1.3.0, rm 1.2.0, am 1.4.0 primary; am 2.4.0, lang 1.1.0, term 3.1.0 deferred) with provenance README.
- BMM loader (`openehr/bmm/`): public `Load` / `LoadAll`, `Resolver`, `FSResolver`, `MapResolver`; full P_BMM persistence model with discriminator-driven decoding and round-trippable polymorphic marshalling; descendant-shadows-ancestor include merge with sibling-conflict detection (REQ-045).
- BMM code generator (`internal/bmmgen/`, `cmd/bmmgen/`): deterministic emission of openEHR RM and AOM 1.4 types from pinned BMM (REQ-042, REQ-043, REQ-044, REQ-046); `-verify` drift detection; `make codegen` / `make codegen-verify`; one-way `aom14 → rm` import sharing base types; mutual-recursion break via `CyclicSingleProps`; method stubs with BMM doc/pre/post propagation.
- BMM version-bump tooling (`cmd/bmmdiff` / `internal/bmmdiff`): semantic diff + CHANGELOG-suggestion helper; weekly drift-detection workflow (`.github/workflows/codegen-drift.yml`); runbook in [`docs/adr/0001-bmm-version-bump-runbook.md`](docs/adr/0001-bmm-version-bump-runbook.md).
- Generated openEHR RM (`openehr/rm/*_gen.go`) and AOM 1.4 (`openehr/aom/aom14/*_gen.go`): ~99 + 39 concrete classes with embedded ancestors and snake_case JSON tags, abstract marker interfaces, generic `Interval[T]` / `DVInterval[T]` / `Event[T]` etc., primitive aliases per [`specs/bmm-conformance.md`](specs/bmm-conformance.md). EHR Extract / foundation builtins / functional packages skipped per spec.
- Type registry (`openehr/rm/typereg/`): goroutine-safe `Registry` with `Default`, panic-on-duplicate `Register`, polymorphic `Decode` / generic `DecodeAs[T]`; sentinel errors `ErrMissingType` / `ErrUnknownType` / `ErrTypeMismatch` and `DecodeError` envelope shared with future canonical-XML (REQ-040, PROBE-031).
- Canonical JSON codec (`openehr/serialize/canjson/`): `Marshal` / `MarshalIndent` / `Unmarshal` / `NewDecoder` / `WithRelaxedTypeDispatch` over generator-emitted per-class `MarshalJSON` (`openehr/rm/*_jsonmar_gen.go`, `openehr/aom/aom14/*_jsonmar_gen.go`) and `UnmarshalJSON` (`*_jsonunmar_gen.go`) — `_type`-first, BMM property declaration order, lexicographic Hash keys, polymorphic dispatch via `typereg.DecodeAs[T]` (REQ-052). Encode-strict, decode-permissive numeric handling for `Real` / `Integer` per ADR 0004. Edge-case tests cover null⇄absent, ISO 8601 passthrough, omit-empty containers, deep FOLDER recursion; benchmarks under `openehr/serialize/canjson/bench_test.go`.
- Vendored canonical-JSON cassettes (`testkit/cassettes/canonical_json/`) with provenance README (REQ-082).
- Conformance probes (`testkit/probes/serialize/`): PROBE-030 (canonical-JSON round-trip stability, exercising leaf types + every vendored cassette) and PROBE-031 (`_type` not in registry wraps `typereg.ErrUnknownType`).
- ADRs: [`docs/adr/0002-bmm-codegen-decisions.md`](docs/adr/0002-bmm-codegen-decisions.md) (codegen layout); [`docs/adr/0003-rm-event-polymorphism.md`](docs/adr/0003-rm-event-polymorphism.md) (promotes abstract generic `EVENT` to a Go interface; closes Event/History sub-question of STRAND-04); [`docs/adr/0004-numeric-wire-tolerance.md`](docs/adr/0004-numeric-wire-tolerance.md) (asymmetric numeric wire tolerance).
- Conformance probes (`specs/conformance.md`): reserves PROBE-060..068 for REST binding (EHR creation, versioned writes with `Prefer`, audit-details, discovery-routed request, per-request `TokenSource` override, stored AQL, template upload, error envelope decode). PROBE-040..049 remain assigned to Service discovery per the spec's no-renumbering rule.
- Vendored REST cassettes (`testkit/cassettes/its_rest/`): openEHR error envelopes (400/401/403/404/409/412/428) per REQ-093 and SMART configuration fixtures (`smart-configuration.json`, `smart-configuration-mismatch.json`, `jwks.json`) with provenance README pinning the upstream ITS-REST OpenAPI commit (REQ-082, REQ-095).
- Authentication foundation (`auth/`): generic `TokenSource` interface, `Token` type, `StaticTokenSource` / `AnonymousTokenSource`, per-request context override (`auth.WithTokenSource` / `TokenSourceFromContext`), sentinels (`ErrTokenExchangeFailed` / `ErrRefreshFailed` / `ErrReauthRequired` / `ErrInvalidConfig` / `ErrJWKSValidationFailed`), `OAuth2Error` + `ExchangeError` wrapper, `BuildScope` / `JoinScopes` helpers (REQ-060, REQ-066, REQ-068).
- Auth providers (`auth/clientcreds/`, `auth/jwtbearer/`): OAuth2 Client Credentials (RFC 6749 § 4.4) with Basic + Post auth methods and JWT Bearer (RFC 7523) `TokenSource` implementations sharing a common pattern — HTTP-client injection, coalesced concurrent refresh, transparent expiry-based refresh, typed `ExchangeError` mapping. `jwtbearer.ClaimsSigner` provides a built-in RS256 signer; `AssertionSource` / `AssertionFunc` / `StaticAssertion` cover externally-minted JWTs.
- Service discovery (`smart/discovery/`): `ServiceCatalog` / `ServiceEntry` / `AuthEndpoints` types, `Resolver` for `.well-known/smart-configuration` fetch + parse + validate + cache + refresh with single-flight coalescing (REQ-026), `Cache` interface + `MemoryCache` default, `NewStaticCatalog` for hand-built catalogs (EHRbase, local CDR), typed `DiscoveryError` with reasons (`fetch_failed` / `parse_error` / `missing_service` / `spec_version_mismatch` / `malformed_url` / `auth_endpoints_missing` / `insecure_url`), strict-by-default spec-version pin (`SpecVersionPin = "1.1.0-development"`) wideable via `WithAcceptedSpecVersions`, plaintext-issuer rejection unless `WithAllowInsecure` and slog-based TLS-posture warnings on resolved catalogs (REQ-070, REQ-071, REQ-072, REQ-092).
- Transport (`transport/`): `Client` (HTTP wrapper around an injected `*http.Client` per REQ-021) with `Do(ctx, *Request) (*Response, error)` and generic `Decode[T any](ctx, c, req) (*T, *Metadata, error)`; typed `Request` carrying ServiceID + Path + Route + Query + Body + Prefer + IfMatch + the full `openehr-*` header family (audit-details, version, template-id, uri, item-tag); typed `Response` + `Metadata` with quote-stripped `ETag` and parsed openEHR/Cadasto headers; functional options (WithHTTPClient, WithTokenSource, WithUserAgent, WithSpecVersion, WithCadastoSpecVersionHeader, WithRetry, WithCallerAttribution, WithCallerAttributionHeader, WithLogger); sentinels (`ErrNotFound`/`ErrUnauthorized`/`ErrForbidden`/`ErrVersionConflict`/`ErrPreconditionFailed`/`ErrPreconditionRequired`/`ErrServerError`/`ErrServiceUnavailable`/`ErrInvalidConfig`/`ErrInvalidShape`) and `WireError` envelope with parsed `OpenEHRErrorDetail` per REQ-093; `Prefer` enum (REQ-094); `RetryPolicy` (off by default, idempotent-only by default, per REQ-091); OTel span emission with `openehr.spec_version` / `http.*` / `retry.*` attributes and W3C `traceparent` / `tracestate` propagation, no-op safe when no `TracerProvider` is registered (REQ-090, PROBE-051); `CallerAttribution` with per-client default and per-request `WithCallerAttributionCtx` override (REQ-066).

### Changed

- Pinned BMM schemas moved from `resources/` into `resources/bmm/`; codegen defaults and docs updated.
- Module path normalized to lowercase `github.com/cadasto/openehr-sdk-go` across code, specs, and docs (REQ-001).
- `specs/wire.md` REQ-052 Floating-point precision section cross-references ADR 0004 for decode tolerance; `specs/bmm-conformance.md` § Primitive type mapping notes `rm.Real` / `rm.Integer` defined types and the `openehr/rm/typereg_gen.go` location per ADR 0002 D3; `specs/rm-modeling.md` § Embedded base structs notes the generator's abstract-flattening rule per ADR 0002 D4.
